## 📝 Quick Summary
This paper addresses a critical flaw in modern AI pipelines: LLMs are notoriously unreliable at generating strictly formatted data (like JSON) that adheres to a predefined schema. Existing solutions usually try to fix this by modifying the LLM's internal decoding engine, which is expensive and locks you into specific models. Instead, the authors introduce **SLOT (Structured LLM Output Transformer)**. SLOT acts as a decoupled, model-agnostic "post-processing layer." It takes the messy, unstructured text generated by a primary LLM and uses a separate, lightweight, fine-tuned model (like Llama-1B or Mistral-7B) whose sole job is to translate that text into perfect JSON. 

## 🎯 Key Findings & Metrics
* **Core Mechanism:** Decoupling the "reasoning task" from the "formatting task." The primary LLM just focuses on generating the right answer in plain text, and the lightweight SLOT layer handles the strict JSON mapping.
* **Main Result:** Using SLOT achieves 99.5% schema accuracy and 94.0% content similarity, massively outperforming state-of-the-art models like Claude-3.5-Sonnet (beating it by +25% in accuracy) when forced to generate structured JSON directly.
* **The Evaluation Metric:** The authors note that traditional "exact string matching" is terrible for evaluating JSON data. Instead, they calculate semantic fidelity using a **Harmonic Mean of Soft-Precision and Soft-Recall** via Sentence-BERT embeddings. This means they mathematically score if the *meaning* of the extracted data matches the ground truth, rather than just checking if the characters match.
