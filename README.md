# Detecting Hallucinations in LLM Summaries of Local Music Venues

> A hybrid evaluation framework for detecting intrinsic and extrinsic hallucinations in LLM-generated concert newsletters.

This project is a research-oriented evaluation framework developed as part of a bachelor's thesis. The project validates automated concert newsletter outputs generated by Large Language Models (LLMs) by combining Natural Language Inference (NLI) with a deterministic music knowledge graph. The proposed two-phase evaluation architecture enables robust detection of both semantic inaccuracies and factual inconsistencies in generated event summaries.

---

## ✨ Features

- **Corpus Builder**
  - Scrapes concert and event data from the Doornroosje music venue website.
  - Builds a structured evaluation corpus for experimentation and benchmarking.

- **Claims Decomposer**
  - Uses spaCy dependency parsing to extract atomic claims from generated summaries.
  - Generates claim profiles without invoking expensive NLI inference.

- **Baseline Evaluator (NLI-Only)**
  - Evaluates intrinsic hallucinations using a cross-lingual XLM-RoBERTa NLI model.
  - Measures semantic consistency between source content and generated summaries.

- **Hybrid Evaluator**
  - Combines NLI-based reasoning with deterministic knowledge graph validation.
  - Detects both intrinsic and extrinsic hallucinations.

- **MusicBrainz Integration**
  - Uses MusicBrainz as an external knowledge source.
  - Provides non-parametric fact verification and override mechanisms.

- **Master Control CLI**
  - Interactive command-line interface for running individual modules or complete workflows.
  - Supports sequential end-to-end execution of the entire pipeline.

- **Test Mode Toggle**
  - Enables fast dry-runs by limiting dataset size and processing volume.
  - Ideal for debugging and development iterations.

## 🚀 Installation & Setup

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/bachelor-thesis.git
cd bachelor-thesis
```

### 2. Create a Virtual Environment

```bash
python -m venv .venv
```

### 3. Activate the Environment

#### Linux / macOS

```bash
source .venv/bin/activate
```

#### Windows

```powershell
.venv\Scripts\activate
```

### 4. Install Dependencies

```bash
pip install -r requirements.txt
```

### 5. Download spaCy Language Model

```bash
python -m spacy download en_core_web_sm
```

### 6. Verify Installation

```bash
python main.py
```

or launch the CLI entry point configured in the repository.

## 💻 Usage Example

### Launch the Interactive CLI

```bash
python main.py
```

### Typical Workflow

```text
1. Build Evaluation Corpus
2. Generate Atomic Claims
3. Run Baseline NLI Evaluation
4. Run Hybrid Evaluation
5. Export Results
```

### Enable Test Mode

```bash
python main.py --test
```

This processes a reduced subset of records for rapid validation and debugging.
